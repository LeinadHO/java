# Parte 3 - Pacotes
- Pacotes são um conjunto de classes agrupadas, normalmente por possuírem algum tipo de relação ou função em comum;
  
- A hierarquia dos pacotes é baseada na estrutura das pastas do projeto.
  
## Parte 3.1 - Convenções de nomenclatura
- Para nomear um pacote deve-se escrever a url do site (invertida) acrescida do nome do projeto;
  
  - Exemplo: **academy.devdojo.maratonajava**, onde *devdojo.academy* é o **nome do site** e *maratonajava* é o **nome do projeto**.

- Para cada nova adição à hierarquia de arquivos, o nome do pacote será extendido para incluir as novas estruturas;

  - Exemplo: **academy.devdojo.maratonajava.introducao**, onde o pacote corresponde ao conjunto das classes criadas durante a parte introdutória do curso.

- **TODAS** as classes desenvolvidas em Java devem pertencer a um pacote;

- Para indicar qual pacote a classe pertence, utiliza-se a palavra reservada **package**, seguida do nome do pacote;

- Obrigatoriamente, a primeira linha do código-fonte deve ser a indicação do pacote.
